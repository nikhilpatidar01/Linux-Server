# 🔐 **Linux `chmod` Command – File Permission Management**  

The `chmod` command in Linux is used to **change file permissions** for users, groups, and others. Below are examples of various `chmod` commands along with their **expected outputs and explanations**.

---

✅ **Correct Syntax:**  
```bash
chmod -R o+w data2
```
🔹 This **adds write permission** for **others** recursively in `data2`.  

---

## 📌 **2. Removing Write Permission for Others (Typo in Command)**  

### ❌ **Input (Typo: `hmod` instead of `chmod`)**
```bash
hmod -R o-w data2
```
### ❌ **Output (Command Not Found)**
```
bash: hmod: command not found
```
✅ **Correct Command:**  
```bash
chmod -R o-w data2
```
🔹 This **removes write permission** for **others** recursively in `data2`.

---

## 📌 **3. Removing All Permissions for Others**  

### ✅ **Input**
```bash
chmod -R o-rwx data2
```
### ✅ **Output (No Output if Successful)**  
🔹 This **removes read, write, and execute permissions** for **others** recursively in `data2`.  
🔹 **Verify with:**  
```bash
ls -l data2
```

---

## 📌 **4. Adding Read, Write, and Execute Permissions for the Owner**  

### ✅ **Input**
```bash
chmod -R u+rwx data2
```
### ✅ **Output (No Output if Successful)**  
🔹 This **grants full access** to the **owner** recursively.  
🔹 **Verify with:**  
```bash
ls -l data2
```

---

## 📌 **5. Adding Read, Write, and Execute Permissions for the Group**  

### ✅ **Input**
```bash
chmod -R g+rwx data2
```
### ✅ **Output (No Output if Successful)**  
🔹 This **grants full access** to the **group** recursively.  
🔹 **Verify with:**  
```bash
ls -l data2
```

---

## 📌 **6. Incorrect Syntax – Missing User Category**  

### ❌ **Input**
```bash
chmod -R 0+rwx data2
```
### ❌ **Output (Error)**
```
chmod: invalid mode: ‘0+rwx’
Try 'chmod --help' for more information.
```
✅ **Correct Command:**  
```bash
chmod -R u+rwx data2
```
🔹 **Must specify a user category** (`u`, `g`, or `o`).

---

## 📌 **7. Adding Write and Execute Permissions for All Users (Typo in Command)**  

### ❌ **Input (`hmod` instead of `chmod`)**
```bash
hmod -R +wx data2
```
### ❌ **Output (Error)**
```
bash: hmod: command not found
```
✅ **Correct Command:**  
```bash
chmod -R +wx data2
```
🔹 This **adds write and execute permissions** for **everyone** recursively.

---

## 📌 **8. Setting Only Write and Execute Permissions for All Users**  

### ✅ **Input**
```bash
chmod -R a=wx data2
```
### ✅ **Output (No Output if Successful)**  
🔹 This **removes all permissions** except **write and execute** for **everyone** in `data2`.  
🔹 **Verify with:**  
```bash
ls -l data2
```

---

## 📌 **9. Removing Write Permission for Group**  

### ✅ **Input**
```bash
chmod -R g-w data2
```
### ✅ **Output (No Output if Successful)**  
🔹 This **removes write permission** for the **group** recursively.  
🔹 **Verify with:**  
```bash
ls -l data2
```

---

## 📌 **10. Adding Write Permission for Group (Incorrect Case)**  

### ❌ **Input (`W` should be lowercase `w`)**
```bash
chmod -R g+W data2
```
### ❌ **Output (Error)**
```
chmod: invalid mode: ‘g+W’
Try 'chmod --help' for more information.
```
✅ **Correct Command:**  
```bash
chmod -R g+w data2
```
🔹 **Permissions are case-sensitive** (`w`, not `W`).

---

## 📌 **11. Removing Execute Permission for Group**  

### ✅ **Input**
```bash
chmod -R g-x data2
```
### ✅ **Output (No Output if Successful)**  
🔹 This **removes execute permission** for the **group** recursively.  
🔹 **Verify with:**  
```bash
ls -l data2
```

---

## 📌 **12. Incorrect Command – Missing Mode**  

### ❌ **Input**
```bash
chmod -R data2
```
### ❌ **Output (Error)**
```
chmod: missing operand after ‘-R’
Try 'chmod --help' for more information.
```
✅ **Correct Command:**  
```bash
chmod -R 755 data2
```
🔹 **Must specify a permission mode** (`777`, `644`, etc.).

---

## 📌 **13. Setting Permissions to `777` (Full Access)**  

### ✅ **Input**
```bash
chmod -R 777 data2
```
### ✅ **Output (No Output if Successful)**  
🔹 **Full permissions** (read, write, execute) for **everyone**.  
🔹 **Verify with:**  
```bash
ls -l data2
```

---

## 📌 **14. Setting Permissions to `754` (More Restrictive)**  

### ✅ **Input**
```bash
chmod -R 754 data2
```
### ✅ **Output (No Output if Successful)**  
🔹 **Owner:** Full permissions (`rwx`)  
🔹 **Group:** Read & execute (`r-x`)  
🔹 **Others:** Read-only (`r--`)  

---

## 📌 **15. Setting Permissions to `700` (Private Access)**  

### ✅ **Input**
```bash
chmod -R 700 data2
```
### ✅ **Output (No Output if Successful)**  
🔹 **Only the owner has full access**.  

---

## 📌 **16. Invalid Mode – Incorrect Octal Code**  

### ❌ **Input**
```bash
chmod -R OOO data2
```
### ❌ **Output (Error)**
```
chmod: invalid mode: ‘OOO’
Try 'chmod --help' for more information.
```
✅ **Correct Command:**  
```bash
chmod -R 700 data2
```
🔹 **Ensure correct numeric format (`0-7`)**.

---

## 📌 **17. Setting `600` (Private Read & Write) for Specific Files**  

### ✅ **Input**
```bash
chmod 600 messages tallytog xrdp.log
```
### ✅ **Output (No Output if Successful)**  
🔹 **Only the owner can read & write** these files.  

---

## 📌 **18. Setting `600` for All Files in a Directory**  

### ✅ **Input**
```bash
chmod 600 *
```
### ✅ **Output (No Output if Successful)**  
🔹 **Applies to all files in the current directory**.  

---

# 📊 **Summary Table of `chmod` Commands**  

| Command | Description |
|---------|------------|
| `chmod -R o-w data2` | Remove write permission for others |
| `chmod -R u+rwx data2` | Grant full access to owner |
| `chmod -R 777 data2` | Give full permissions to all users |
| `chmod -R 700 data2` | Only owner has full access |
| `chmod -R g+w data2` | Add write permission for group |
| `chmod -R g-x data2` | Remove execute permission for group |
